# 一、项目概述

## 1. 项目简介

### 1.1 项目名称

本项目的名称为 **Zchoose-AI 穿搭助手**。  
本项目围绕日常穿搭与试衣场景，为用户提供推荐、试衣与闲置衣物流转等服务。  
衣库除日常、通勤、约会等常见场合外，还涵盖**星座专属穿搭**、**民族风穿搭**、**古装**、**二次元穿搭**、**时装秀场**等多元风格，既能提升用户的使用兴趣，也与普通服饰购物形成明显区分。

### 1.2 平台

本项目目前以 **网页端** 为主。  
前端使用 React 与 Vite 搭建。  
后端使用 Node.js、Express 与 SQLite（通过 sql.js）实现接口与数据存储。  
前端默认访问地址为 `http://localhost:5173`，后端默认访问地址为 `http://localhost:3001`。  
项目已接入 Cloudflare Tunnel，并将域名 `z-choose.com` 绑定到本机服务；在隧道与前后端均运行的情况下，用户可通过 `https://z-choose.com` 访问前端。  
底部导航与左侧导航中均设有「衣库」「闲置」「快速穿搭」「客服」「我的」等入口。

### 1.3 理念

本项目的理念主要有四点。  
第一，以用户本人为模特，让试衣效果围绕用户自身形象展开，而不是仅展示商家模特图。  
第二，将穿搭决策拆成季节、场合、性别、年龄段等简单维度，通过标签筛选与推荐降低选择成本。  
第三，在推荐与积分等规则上尽量透明，使用户能够理解「为什么看到这些搭配」以及「如何解锁更多内容」。  
第四，通过星座、民族风、古装、二次元、时装秀场等多元风格，让普通用户也能低成本体验秀场感、古风与二次元等穿搭，既增强使用兴趣，也与「只卖日常款」的普通购物形成区分。

### 1.4 服务

本项目目前已实现或规划的服务如下。

（1）**穿搭推荐服务**  
后端根据天气、季节及用户在「我的」页设置的偏好（如性别、年龄段、场合），从衣库中筛选搭配。  
用户可选择场合（如日常、通勤、约会、运动、过年）与季节（春、夏、秋、冬），并可设置偏好性别与年龄段（少年、青年、中年、老年）。  
首页与推荐页以卡片形式展示多套搭配，用户可刷新或随机抽取。

（2）**AI 试衣服务**  
用户上传个人照片并在衣库中选择一套搭配后，后端通过配置的试衣服务地址请求生成三视图试衣图。  
若未配置真实试衣服务，则返回占位图；若已配置，则返回实际生成效果。  
用户可在试衣结果页查看并下载图片；试衣等待期间提供小游戏与倒计时。

（3）**衣库与标签筛选服务**  
后端扫描 `frontend/public/images` 下按场合、季节、性别、年龄段命名的目录，将图片信息写入 `outfits` 表并打上对应标签。  
前端衣库页支持按风格、季节、性别、年龄段四类标签组合筛选；风格除日常、通勤、约会、运动、过年外，还包括**星座专属穿搭**、**民族风穿搭**、**古装**、**二次元穿搭**、**时装秀场**等，满足用户对多元穿搭的体验需求。  
衣库以卡片形式展示搭配，支持点击放大图片与跳转试衣。后期可引入真实民族服饰、秀场款与古装/二次元周边等实物，用户既可纯体验，也可在体验后通过商家槽位或链接完成购买。

（4）**积分与解锁服务**  
数据库中设有 `user_points`、`user_unlocks`、`user_outfit_likes` 等表。  
部分衣库搭配被设为「需积分解锁」，用户解锁后可试穿。  
用户通过投稿被采纳或完成指定行为获得积分；前端在衣库卡片上展示「喜欢」与解锁状态。

（5）**商家入驻与尺码匹配服务（基础版）**  
数据库中设有 `merchants`、`outfit_merchant_slots`、`merchant_size_rules` 等表。  
每套搭配可为「上衣 / 裤子 / 鞋子」三个槽位配置商家与商品链接。  
前端提供商家入驻槽位管理页；示例数据中包含按身高体重区间匹配尺码的规则。

（6）**闲置衣物流转与商家过季专区**  
数据库中新增 `resale_items` 表，用于记录用户发布的闲置衣物与商家发布的过季货。  
每条记录包含：发布人、来源类型（用户闲置或商家过季）、标题、描述、图片地址、季节/场合/性别/年龄段标签、价格、占位费（当前为 2 元/套，仅记录、未实际扣费）、状态（在售、下架、已售、草稿）等。  
后端提供 `/api/resale-items` 的增删改查及状态更新接口；列表接口支持按来源类型、标签及价格区间（0–200、200–500、500–1000、1000–10000、奢侈品）筛选。  
前端设有「闲置」页（`/resale`），可根据用户身份展示「发布闲置衣物」或「发布商家过季」入口；发布页（`/resale/new`）包含费用说明（2 元/套）、表单与「返回闲置市场」按钮。  
用户与商家均可在此发布并筛选流转衣物。

（7）**会员与免费试穿配额说明**  
会员档位为月卡、季卡、年卡，价格分别为 10 元、35 元、100 元。  
在「我的」页中已更新文案：月卡用户当期内可免费试穿 5 套积分解锁衣物，季卡 15 套，年卡 50 套。  
当前后端尚未对上述配额做严格计数与校验，仅在前端完成规则说明与展示。

### 1.5 创新点

本项目在现有实现范围内的创新点主要包括：将 AI 试衣与日常穿搭决策结合；用标签化推荐（场合、季节、性别、年龄）使推荐逻辑可理解；通过积分解锁与会员免费试穿配额设计激励长期使用；预留商家入驻与尺码匹配能力；新增闲置与过季衣物流转及价格分层筛选；结合 Cloudflare Tunnel 与自定义域名实现本地开发环境下的外网访问。  
此外，衣库支持**星座专属**、**民族风**、**古装**、**二次元**、**时装秀场**等多元风格，既显著提升用户使用兴趣，也与普通服饰电商形成明显区分。后期引入真实民族服饰，有助于各国传统服饰文化的展示与传承；时装秀场、古装与二次元等风格既可纯体验，也可通过对接真实服装实现「体验后购买」，满足单纯尝鲜与真实购衣两种需求。

## 2. 项目背景

### 2.1 宏观层面

从整体环境看，人们对日常形象与穿搭的关注度在提升。  
社交媒体与短视频放大了穿搭话题的曝光，许多用户每天都会面对「今天穿什么」的问题。  
同时，AI 图像生成能力快速发展，使个性化试衣在技术上更具可行性。  
本项目在此背景下，尝试将 AI 能力与日常穿搭需求结合，做成一个可用的产品形态。

### 2.2 微观层面

从个人与小组角度看，项目源于发起人对日常选衣的困扰。  
早期先实现了「上传照片 + 选择搭配 + 生成三视图」的试衣流程。  
随后发现仅试衣不够，用户在选择前仍会纠结，因此增加了按天气与场合的推荐、衣库筛选与快速穿搭等功能。  
之后在衣库中陆续增加**星座专属**、**民族风**、**古装**、**二次元**、**时装秀场**等风格标签，以增强用户兴趣并与普通购物形成区分；又根据需求增加了闲置衣物流转与商家过季专区、价格分层与身份区分的发布入口，以及会员免费试穿配额的文案设计。  
开发过程中持续调整数据表、接口与页面，使功能与文档描述保持一致。

### 2.3 政策层面

本项目目前主要在本地与小范围演示环境运行，未接入真实支付与大规模用户数据。  
已实现部分与合规相关的设计：用户上传图片与账号关联并做访问控制；试衣与推荐逻辑不依赖未告知的数据采集。  
若后续上线或扩大使用范围，需在个人信息保护、内容审核与数据存储等方面按当时法规与平台要求补充与调整。

### 2.4 行业层面

服饰电商与虚拟试衣领域已有不少产品，有的侧重商品展示，有的侧重单点试衣技术或转化。  
本项目规模较小，功能相对集中，但在以下方面做了区分：强调以用户本人为中心组织试衣与推荐；将推荐与衣库按场合、季节、年龄等标签拆解，便于理解；预留商家与尺码能力；新增闲置与过季流转及价格分层，与「减少闲置、理性购衣」的方向衔接。  
衣库涵盖**星座专属**、**民族风**、**古装**、**二次元**、**时装秀场**等多元风格，让普通用户也能低成本体验秀场款、古装与二次元等穿搭；后期引入真实民族服饰可助力各国传统服饰文化的展示与传承，秀场、古装与二次元等风格也可通过对接真实服装，同时满足「纯体验」与「体验后购买」两种需求。

## 3. 项目优势

### 3.1 创新点

本项目将 AI 试衣、场景化推荐、标签筛选、积分解锁、会员免费试穿说明、闲置与过季衣物流转及价格分层整合在一起。  
衣库中的**星座专属**、**民族风**、**古装**、**二次元**、**时装秀场**等多元风格，既能提升用户使用兴趣，也与普通服饰购物形成明显区分；后期引入真实民族服饰可助力文化传承，秀场、古装与二次元等既可纯体验，也可对接真实服装实现购买。  
用户可以从推荐或衣库进入试衣，也可以从闲置市场发布或筛选衣物；逻辑清晰，便于后续扩展与文档说明。

### 3.2 模式

本项目先做「穿搭助手 + 试衣 + 衣库」，再逐步接入商家与闲置流转。  
数据库中已具备商家、衣库槽位、闲置与过季记录等结构，为后续扩展留出空间，便于小团队分阶段推进。

### 3.3 技术
前端采用 React 与 Vite，组件按页面划分，便于维护。  
后端采用 Express 与 sql.js，单进程即可完成接口与 SQLite 读写。  
衣库通过扫描目录与标签写入数据库，减少手工录入。  
试衣通过配置的外部地址调用，与主站解耦。  
闲置与过季接口支持按类型、标签与价格区间筛选。  
通过 Cloudflare Tunnel 与自定义域名可在本地环境下实现外网访问与演示。

### 3.6 社会效益

项目有助于用户减少穿搭决策成本与试错成本，并在文案与设计中体现「按场合、按年龄选衣」及「闲置流转、价格分层」等理性购衣与减少浪费的导向。  
多元风格（民族风、古装、星座、二次元、时装秀场）让普通用户也能体验不同文化与时序下的穿搭，后期引入真实民族服饰有助于各国传统服饰文化的展示与传承；秀场、古装与二次元等既可纯体验，也可通过真实服装购买落地，兼顾兴趣满足与理性购衣。  
若后续与更多商家或公益活动结合，可进一步强化社会效益部分的表述与落地。
