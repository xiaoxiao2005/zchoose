# 试衣「生成失败」排查说明

试衣时出现「生成失败」或类似提示，可按下面顺序排查。

---

## 一、看具体提示文案

- **「今日免费试衣次数已用完」** → 当天 5 次已用尽，明天再试或联系客服。
- **「请先使用积分解锁该搭配」** → 当前选的是星座专属/个性化搭配，需在衣库中先消耗积分解锁后再试衣。
- **「无权使用该照片试衣」** → 使用的照片不是本人上传的，请换成本人上传的照片。
- **「需要 userId, outfitId, photoUrl」** → 请求参数不完整，多为前端异常，可刷新页面重试。
- **「登录已过期，请重新登录」** → 重新登录后再试。
- **仅显示「生成失败」或「试衣生成异常」** → 见下文二、三。

---

## 二、确认后端与试衣服务是否在跑

试衣流程：**前端 → 后端（Node）→ tryon-service（Python，若已配置）**。任一层未启动或不可达，都可能报错。

### 1. 后端（Node）必须一直运行

- 本地开发：在项目里执行 `cd backend && npm run dev`，保持终端不关。
- 线上：确保 Node 进程在跑（如 PM2 `pm2 list` 里 zchoose-api 为 online）。

若后端没跑，前端请求会**连不上**或**超时**，容易出现「生成失败」或网络错误。

### 2. tryon-service（Python）决定是否有真实试衣图

- 未配置或未启动时：后端会返回**占位图** + 说明文案（如「请配置 TRYON_API_URL 并启动 tryon-service」），一般不会直接报「生成失败」；若仍报「生成失败」，多半是**后端整体未响应**（见上一条）或**后端抛错**（见第三节）。
- 需要真实试衣效果时：
  - 在 **tryon-service** 目录执行：`python main.py`（或你实际使用的启动命令），保持运行。
  - 在 **backend** 的 `.env` 里配置：`TRYON_API_URL=http://localhost:8000/generate`（端口与 tryon-service 一致）。
- 若 tryon-service 崩了或返回错误，后端会退回占位图并带 hint，通常不会给前端 4xx/5xx，所以**纯「生成失败」多数不是 tryon-service 单独的问题**，而是**请求没到后端**或**后端抛异常**。

---

## 三、如何确认是「请求没到后端」还是「后端报错」

1. 打开浏览器 **F12 → Network（网络）**，再点一次试衣生成。
2. 找到 **`generate`** 或 **`try-on`** 的请求，看：
   - **状态码**：
     - **红色 / failed / 无响应**：请求没到后端或超时 → 检查后端是否启动、地址/端口是否正确（本地是否 3001、线上是否 Nginx 转发到 3001）。
     - **403**：今日次数用完、未解锁搭配、或无权使用该照片 → 看响应里 `error` 字段的文案。
     - **404**：搭配不存在等 → 看响应里 `error`。
     - **500**：后端内部错误 → 看服务器上 backend 的日志（如 `console.error('[try-on/generate]', e)` 的输出），或响应里的 `error` 字段。
   - **Response（响应）**：若为 JSON，里面的 `error` 会说明原因。

---

## 四、小结

| 现象 | 优先检查 |
|------|-----------|
| 提示「今日免费试衣次数已用完」 | 当天 5 次已用尽，次日再试 |
| 提示「请先使用积分解锁该搭配」 | 在衣库中对该搭配使用积分解锁后再试衣 |
| 提示「无权使用该照片试衣」 | 使用本人上传的照片 |
| 仅「生成失败」且 Network 里请求失败/无响应 | 后端未启动或不可达：先保证 backend 已跑，再试 |
| 仅「生成失败」且请求 500 | 看后端日志或响应里的 error，再查 tryon-service 是否正常、数据库是否正常 |

若按上面仍无法解决，请把**浏览器 Network 里该请求的状态码、响应 body（或后端日志里报错的那几行）**提供给排查人员，便于精确定位。
