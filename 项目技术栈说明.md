# 项目技术栈说明

本文档介绍本穿搭试衣与推荐项目所涉及的技术选型、架构与关键依赖，便于开发与维护时查阅。

---

## 一、整体架构

项目采用**前后端分离**结构：

- **前端**：单页应用（SPA），运行在浏览器，通过 HTTP 请求与后端交互。
- **后端**：Node.js 服务，提供 REST API、静态资源与数据库读写。
- **数据**：SQLite 单文件数据库（通过 sql.js 在 Node 中运行），数据持久化在 `backend/data/app.db`。
- **可选外部服务**：试衣生成 API（tryon-service）、天气 API（和风 / Open-Meteo）。

开发环境下，前端由 Vite 在端口 5173 启动，并将 `/api`、`/uploads`、`/images` 代理到后端 3001 端口，实现同源访问。

## 二、前端技术栈

### 2.1 核心框架

| 技术 | 版本 | 说明 |
|------|------|------|
| **React** | ^18.2.0 | 用于构建 UI 组件与页面，采用函数组件 + Hooks 写法。 |
| **React DOM** | ^18.2.0 | React 的 DOM 渲染层，将组件树渲染为浏览器 DOM。 |
| **React Router DOM** | ^6.20.0 | 前端路由：处理 `/home`、`/tryon`、`/me` 等路径与布局嵌套（如 `Layout` 包裹多页）。 |

### 2.2 开发与构建

| 技术 | 版本 | 说明 |
|------|------|------|
| **Vite** | ^5.0.8 | 构建与开发服务器。开发时提供 HMR、ES 模块原生支持；生产构建使用 Rollup 打包，输出到 `dist`。 |
| **@vitejs/plugin-react** | ^4.2.1 | Vite 的 React 插件，支持 JSX/TSX 与 React Fast Refresh。 |
| **TypeScript** | ^5.3.2 | 类型检查与 TS/TSX 编译，前端代码全部为 `.ts` / `.tsx`。 |

### 2.3 前端工程配置要点

- **模块系统**：`package.json` 中 `"type": "module"`，使用 ES Module。
- **脚本**：`npm run dev` 启动 Vite 开发服务器；`npm run build` 先执行 `tsc -b` 再 `vite build`。
- **代理**（`vite.config.ts`）：将 `/api`、`/uploads`、`/images` 代理到 `http://localhost:3001`，便于本地联调且无需 CORS 处理。
- **静态资源**：`frontend/public/` 下文件（如图片、音效）在开发与生产环境中按根路径提供（如 `/images/...`）。

### 2.4 前端技术点摘要

- **状态管理**：以 React 本地 state + Context（如 `AuthContext`）为主，无 Redux 等全局库。
- **请求**：使用原生 `fetch` 调用后端 API，鉴权通过 `Authorization: Bearer <token>` 传递。
- **样式**：各页面/组件对应独立 `.css` 文件，采用 BEM 风格类名（如 `tryon__preset-bg`），并通过媒体查询做移动端适配。
- **路由结构**：根路由重定向到 `/splash`；主内容在 `Layout` 下，包含首页、衣库、试衣、推荐、客服、我的、商家槽位管理等页面。

## 三、后端技术栈

### 3.1 运行环境与语言

| 技术 | 版本 | 说明 |
|------|------|------|
| **Node.js** | 建议 18+ | 服务端运行时，执行编译后的 `dist/index.js`。 |
| **TypeScript** | ^5.3.2 | 后端源码为 `.ts`，编译为 `.js` 到 `backend/dist`。 |

### 3.2 核心框架与中间件

| 技术 | 版本 | 说明 |
|------|------|------|
| **Express** | ^4.18.2 | Web 框架：注册路由、静态资源、全局错误处理、请求体解析等。 |
| **cors** | ^2.8.5 | 跨域中间件，允许前端域名访问后端 API。 |
| **express.json()** | 内置 | 解析请求体为 JSON。 |
| **multer** | ^1.4.5-lts.1 | 处理 `multipart/form-data` 上传（用户头像、试衣照片、背景图、客服图片等），限制单文件 10MB。 |

### 3.3 数据库

| 技术 | 版本 | 说明 |
|------|------|------|
| **sql.js** | ^1.10.0 | SQLite 的 JavaScript 实现（基于 WebAssembly），在 Node 中无需单独安装 SQLite 二进制。数据库文件为 `backend/data/app.db`，启动时加载到内存，请求结束后可写回文件（`saveDb()`）。 |

数据库由 `backend/src/db/init.ts` 统一初始化：建表、迁移（如 `ALTER TABLE` 增加字段）、扫描 `frontend/public/images` 下的衣库目录并写入 `outfits` 等。主要表包括：`users`、`outfits`、`tryon_results`、`support_messages`、`outfit_merchant_slots`、`douyin_claims` 等。

### 3.4 认证与安全

| 技术 | 版本 | 说明 |
|------|------|------|
| **jsonwebtoken** | ^9.0.2 | 签发与校验 JWT，用于登录态；前端将 token 放在请求头或本地存储。 |
| **bcryptjs** | ^2.4.3 | 用户密码哈希存储与校验，注册/登录时使用。 |

### 3.5 配置与环境变量

| 技术 | 版本 | 说明 |
|------|------|------|
| **dotenv** | ^16.3.1 | 从 `backend/.env` 加载环境变量，如 `PORT`、`DATABASE_PATH`、`TRYON_API_URL`、`WEATHER_API_KEY` 等。 |

### 3.6 后端工程配置要点

- **入口**：`backend/src/index.ts`，挂载所有 API 路由（`/api/health`、`/api/users`、`/api/try-on` 等）和 `/images` 静态服务（指向 `frontend/public/images`）。
- **脚本**：`npm run dev` 使用 `tsx watch` 监听源码并运行；`npm run build` 为 `tsc` 编译；`npm run start` 运行 `node dist/index.js`。
- **监听地址**：服务监听 `0.0.0.0:PORT`，便于局域网访问。

## 五、外部服务与集成

### 5.1 试衣生成（可选）

- **环境变量**：`TRYON_API_URL`（如 `http://localhost:8000/generate`）。
- **说明**：未配置时，试衣接口返回占位图；配置后，`tryOnGenerate` 服务将用户照片、搭配图、身材描述等 POST 到该 URL，期望返回 `front_url`、`side_url`、`back_url`。
- **对接方**：可与独立 tryon-service（如 Python 服务）对接，实现真实 AI 试衣。

### 5.2 天气接口

- **可选**：`WEATHER_API_KEY`、`WEATHER_PROVIDER`（如 `qweather`）、`WEATHER_API_URL`。
- **默认**：使用 Open-Meteo 免费 API（地理编码 + 预报），无需 key。
- **用途**：快速穿搭推荐页根据城市与天气给出建议。

## 八、文档与扩展

- 更多业务与接口说明见项目内其他文档（如登录注册、试衣 API 对接、部署说明等）。
