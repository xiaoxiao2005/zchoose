# 真机测试清单（试衣模块）

> 目标：验证“上传人像 -> 选择衣物 -> 预览放大 -> 确认选择 -> 生成试衣 -> 查看/下载结果 -> 结果落盘”全链路可用。

## 一、环境与服务检查（P0）

- [ ] 手机与开发机在同一网络，可互相访问
- [ ] 前端页面可在手机浏览器正常打开
- [ ] `backend` 服务正常运行
- [ ] `tryon-service` 服务正常运行（端口可访问）
- [ ] FASHN 配置有效（`FASHN_API_KEY` 或 `FASHN_VTON_URL`）
- [ ] 结果目录存在且可写：`D:\cursor\tryon-service\result`

## 二、主流程冒烟（P0）

- [ ] 登录后进入试衣页面成功
- [ ] 上传人像成功（含预览）
- [ ] 官方衣库点击某套衣服后，弹出放大预览
- [ ] 预览点“返回”后，不会误选
- [ ] 再次预览点“确定”后，已选状态正确
- [ ] 点击“生成”后有加载态，且不会重复触发
- [ ] 生成成功后可查看结果图
- [ ] 下载图片成功
- [ ] 每次生成后，`D:\cursor\tryon-service\result` 新增一张结果图

## 三、我的衣库流程（P0/P1）

- [ ] “上传到我的衣库”成功，并出现在列表里
- [ ] 点击我的衣物后，出现放大预览
- [ ] 预览“确定/返回”行为正确
- [ ] 选中我的衣物后可正常生成试衣图
- [ ] 退出页面再次进入，“我的衣库”数据仍可读取

## 四、异常场景（P1）

- [ ] 未登录时上传我的衣物，提示正确
- [ ] 网络中断时上传/生成，提示正确
- [ ] 图片质量过低时，提示正确
- [ ] 生成服务超时时，提示正确
- [ ] 连续点击生成按钮，不会重复创建任务

## 五、移动端体验（P1）

- [ ] 小屏手机（约 360~430 宽）布局不破
- [ ] 预览弹层完整显示，按钮可点击
- [ ] 页面滚动、弹层滚动体验正常
- [ ] 文字提示无截断、无遮挡

## 六、验收标准（建议）

- 主流程连续通过 3 次以上
- 我的衣库上传 + 选择 + 生成连续通过 3 次以上
- 错误提示可理解、可恢复
- 结果目录落盘与前端显示一致
