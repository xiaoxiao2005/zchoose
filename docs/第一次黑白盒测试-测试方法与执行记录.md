# 第一次黑白盒测试-测试方法与执行记录

## 1. 测试目标

- 验证新增“我的衣库 + 用户上传衣物试穿”功能可用性。
- 验证关键异常分支（鉴权、越权、参数校验、低质量图拦截）是否生效。
- 输出可复用的第一版测试方法，便于后续回归。

## 2. 测试范围

- 后端接口：
  - `POST /api/users/send-code`
  - `POST /api/users/register`
  - `POST /api/upload/photo`
  - `POST /api/wardrobe`
  - `GET /api/wardrobe/my`
  - `DELETE /api/wardrobe/:id`
  - `POST /api/try-on/generate`
- 前端改动点（本轮以接口与逻辑验证为主）：
  - `我的 > 我的衣库` 上传入口
  - `试衣页` 我的衣库上传与选择
  - 图片质量检测逻辑（代码路径检查 + 联动流程验证）

## 3. 测试环境

- OS: Windows 10
- 后端服务：`http://localhost:3001`（运行中）
- 数据库：SQLite(sql.js)，使用当前开发库
- 试衣服务：已配置 `TRYON_API_URL`
- 测试方式：接口自动化脚本（Node 单次脚本）+ 白盒分支接口验证

## 4. 测试方法（可复用）

### 4.1 黑盒测试方法

1. 准备测试账号（动态手机号，避免污染已有账号）。
2. 走完整用户路径：
   - 获取验证码
   - 注册
   - 上传图片
   - 存入我的衣库
   - 查询我的衣库
   - 发起试穿（使用 `wardrobeItemId`）
3. 关注关键断言：
   - HTTP 状态码是否符合预期（200/201）
   - 返回结构字段是否完整（如 `resultId`、`result_url`、`tryonRemaining`）

### 4.2 白盒测试方法

针对关键分支定向构造输入，逐条验证：

- 非上传路径图片入库拦截（`image_url` 非 `/uploads/`）
- 无效 token 访问我的衣库
- 试穿缺少 `outfitId/wardrobeItemId`
- 越权使用他人衣物试穿
- 越权删除他人衣物

### 4.3 前端手工验证建议（本轮后续补充）

- 在“我的衣库”上传低质图，确认质检拦截文案可读。
- 在“我的衣库”列表确认质量分与“建议重拍”标签展示。
- 在“试衣页”切换“官方衣库/我的衣库”，验证交互无冲突。

## 5. 执行记录与结果

## 5.1 黑盒主流程结果

- `send-code`：200（通过）
- `register`：201（通过）
- `upload/photo`：200（通过）
- `wardrobe save`：201（通过）
- `wardrobe list`：200（通过，返回数量正确）
- `try-on generate`：201（通过，返回结果结构完整）

说明：接口链路整体可跑通，新增能力已具备“可用”状态。

## 5.2 白盒关键分支结果

- 非 `/uploads/` 图片地址保存衣库 -> 400（通过）
- 使用无效 token 访问我的衣库 -> 401（通过）
- 试穿不传 `outfitId/wardrobeItemId` -> 400（通过）
- 使用他人 `wardrobeItemId` 试穿 -> 403（通过）
- 删除他人衣物 -> 404（通过）

说明：鉴权与越权防护、参数校验分支生效。

## 6. 结论

- 新增“我的衣库 + 试穿”功能在接口层面已达到第一轮上线前可测状态。
- 发现 1 个高优先级问题（详见问题报告）：试衣服务回拉用户上传图时出现 404，导致返回占位图而非真实试穿图。

## 8. 修复后复测（增量）

- 已实施修复：`/api/try-on/generate` 对衣物图（`outfitImageRef`）若为 `/uploads/...`，统一改为带 token 的受控访问 URL（`/api/upload/access/:filename?token=...`）。
- 复测结果：
  - 原 404 提示（`/uploads/... Not Found`）不再出现，说明代码路径修复生效。
  - 当前返回为 tryon-service 上游 `502`（环境/服务可用性问题），仍回退占位图。
- 结论：
  - **代码缺陷已修复**（P1-001 关闭）；
  - **环境依赖问题待处理**（需检查 tryon-service 可用性与配置）。
