# 试衣与健康管理需求摘要

## 一、健康管理 / 体型评估

### 1.1 输入
- 用户已选：**身材类型**（梨型、沙漏型等）、**身高**、**体重**（与当前「我的」- 体型与身体数据一致）

### 1.2 交互与输出
- 在试衣界面提供 **「一键评估」**（或类似）按钮
- 点击后输出：
  - **体型结论**：偏瘦 / 偏胖 / 正常（可根据 BMI 或自定义规则计算）
  - **推荐尺码**：如 S / M / L / XL 等（先按规则或简单表计算，后期可接入商家尺码表）

### 1.3 后期扩展（商家合作）
- 引入商家 **尺码参数**：如「多少斤穿多少尺码」、身高区间对应尺码
- 推荐尺码可结合：当前体型结论 + 商家尺码表

---

## 二、试衣流程简化

### 2.1 当前流程（可保留为选项）
- 上传人物照 → 选穿搭 → 生成 **三视图**（正面 / 侧面 / 背面）

### 2.2 新流程（主流程）
- 用户上传 **全身图** 或 **半身图**
- 用户选择 **衣库中的服装**（一套）
- 点击 **「生成」** → 只生成 **一张试衣图**（不再强制三视图）
- 后端 / tryon-service 需支持：只返回一张图（如 `result_url` 或沿用 `front_url` 单图）

---

## 三、替换背景（可选功能）

### 3.1 能力
- 在试衣结果基础上，增加 **「替换背景」** 选项
- 用户可选择新背景（上传背景图或选预设背景）
- 将 **人物（含试衣结果）与背景融合**，输出一张合成图

### 3.2 交互
- 试衣生成一张图后，提供 **「换背景」/「不换背景」** 选择
- 选「换背景」：再选/传背景 → 生成融合图并输出
- 选「不换背景」：直接使用当前试衣图

---

## 四、实现顺序建议

| 阶段 | 内容 | 说明 |
|------|------|------|
| 1 | 体型结论 + 推荐尺码（试衣页展示） | 用身高体重算 BMI → 偏瘦/偏胖/正常；推荐尺码先做简单规则或本地表 |
| 2 | 试衣只生成一张图 | 前端改为只展示一张图；后端/tryon 返回单图即可 |
| 3 | 替换背景 | 接入人像抠图 + 背景融合（可对接现有 API 或自建） |
| 4 | 商家尺码表 | 与商家合作后，把「体重/身高 → 尺码」参数接入，替换或补充当前推荐尺码逻辑 |

---

## 五、与现有代码的对应关系

- **身材类型、身高、体重**：已存在于「我的」- 体型与身体数据，以及试衣页表单项；可复用。
- **试衣生成**：当前为三视图（front_url / side_url / back_url）；改为单图时，可保留三视图为可选，主流程只请求/展示一张图。
- **替换背景**：需新增接口与前端步骤（上传/选背景 → 调用融合服务 → 展示结果）。

此文档可作为产品与开发对齐用；实现时按「体型评估 → 单图试衣 → 换背景 → 商家尺码」顺序推进即可。
