# 让他人访问本项目的网址

## 一、同一局域网内（如家里 WiFi、公司内网）

当前前后端已配置为**允许本机以外访问**：

- **后端**：监听 `0.0.0.0:3001`，局域网内可访问。
- **前端**：Vite 已设置 `host: true`，监听 `0.0.0.0:5173`，局域网内可访问。
- **前端请求**：`/api`、`/uploads`、`/images` 通过 Vite 代理到本机 3001，他人只需访问「前端地址」即可，无需单独访问后端。

### 操作步骤

1. **启动服务**
   - 先启动后端：`cd backend && npm run dev`（或 `node dist/index.js`）。
   - 再启动前端：`cd frontend && npm run dev`。

2. **获取本机 IP（给他人用的地址）**
   - 启动后端后，终端会打印**局域网访问**地址，例如：
     - `前端: http://192.168.1.100:5173（需先 cd frontend && npm run dev）`
     - `后端: http://192.168.1.100:3001`
   - 若未看到，可手动查 IP：
     - **Windows**：命令提示符输入 `ipconfig`，看「IPv4 地址」。
     - **Mac / Linux**：终端输入 `ifconfig` 或 `ip addr`，看对应网卡的 `inet` 地址。

3. **把网址发给他人**
   - 对方需与你在**同一 WiFi 或同一内网**。
   - 在浏览器打开：**http://你的IP:5173**（例如 `http://192.168.1.100:5173`）。
   - 对方用该地址即可使用，效果与本机访问 `http://localhost:5173` 一致。

4. **若无法访问**
   - 确认本机**防火墙**已放行 5173、3001 端口。
   - **Windows**：「Windows  Defender 防火墙」→「高级设置」→「入站规则」→ 新建规则，允许 TCP 端口 5173 和 3001。

## 二、互联网上任意人访问（外网用户）

本机通常没有公网 IP，外网无法直接通过「你的内网 IP」访问。需要**内网穿透**或**部署到服务器**。

### 方式 A：Cloudflare Tunnel（推荐，免费 HTTPS）

1. 安装：`winget install Cloudflare.cloudflared`（Windows）。
2. 先启动本项目后端 + 前端（见上文「启动服务」）。
3. 新开终端执行：`cloudflared tunnel --url http://localhost:5173`。
4. 终端会输出 **https://xxx.trycloudflare.com**，把该网址发给他人即可访问。
5. 详细步骤见 **《Cloudflare-Tunnel-使用说明.md》**。

### 方式 B：其他内网穿透（ngrok / cpolar 等）

1. 使用 [ngrok](https://ngrok.com/) 或 [cpolar](https://www.cpolar.com/) 等工具。
2. 安装后执行（以 ngrok 为例）：`ngrok http 5173`。
3. 工具会给出一个**公网网址**（如 `https://xxxx.ngrok.io`），把该网址发给别人即可在浏览器打开。
4. 说明：所有接口（含 `/api`、用户上传访问等）都经前端 5173 的 Vite 代理到本机 3001，**只需暴露 5173**，无需单独暴露 3001。

### 方式 C：部署到云服务器

将前端打包后与后端一起部署到有公网 IP 的服务器，并配置 Nginx 反向代理（80/443 转发到前端静态与后端 API）。详见 **《部署与安全说明.md》**。

---

**总结**：同一 WiFi/局域网内用 **http://本机IP:5173**（启动后端时终端会提示）；外网访问推荐 **Cloudflare Tunnel**（`cloudflared tunnel --url http://localhost:5173`），或 **ngrok/cpolar** 等内网穿透，或 **部署到服务器**。
