# 激励制度与盈利方案（单独文档）

> 本文档解决：**试衣成本 0.55 元/张不亏本**、**与多少商家合作、如何合作**、**时尚能量 / 连续登录 / 积分与产品操作的结合**、**在「不按次向用户售卖试衣」前提下如何主要靠 B 端与用量规则盈利**，以及**对《项目计划书》第七部分的修改建议**。  
> **补充（与主计划书第七部分对齐）：** 产品已上线 **C 端会员在线支付（Stripe）**，权益为**积分解锁类搭配的当期免费解锁次数**，试衣生成仍受时尚能量等约束；下文「不向用户收费」宜理解为**不将试衣按张明码标价售卖**，而非排除会员增值。  
> 与主计划书配套使用，答辩或与导师沟通时可单独引用。

## 一、成本约束与「不亏本」前提

### 1.1 刚性成本

- **单次试衣生成成本：0.55 元/张**（当前按 FASHN API 口径）。
- 成本随调用次数线性增长，**每多生成一张图就多 0.55 元**。
- **不亏本原则**：每一张图对应的收入（或等效价值）≥ 0.55 元，或**总试衣量被规则严格约束**，使总成本不超过可获得的收入。

### 1.2 两种兜底方式

| 方式 | 说明 |
|------|------|
| **收入覆盖** | 每张图都有对应收入 ≥ 0.55 元（主要来自 B 端商家），平台不贴钱。 |
| **用量受控** | 用「时尚能量」等规则限制每用户每日/每周期试衣次数，使总调用量可控，再通过 B 端收入整体覆盖这部分成本。 |

实际采用**两者结合**：用量由激励规则控制，收入主要由 B 端合作覆盖，**不依赖向用户收费**。

## 二、商家合作：10 家 vs 20 家怎么合作

核心思路：**试衣成本由「被试穿款式所属商家」对应的 B 端收入覆盖**。用户试穿某商家的款，该次 0.55 元成本由该商家通过合作费/按量费覆盖。

### 2.1 合作方式概览

| 项目 | 10 家商家 | 20 家商家 |
|------|-----------|------------|
| **定位** | 试点、验证单位经济性 | 放量、摊薄单家压力 |
| **单家负担** | 相对高（试衣量/费用占比大） | 相对低（量分散） |
| **平台收费方式** | 更适合「保底 + 按量」或纯按量 | 可「包月 + 超额按量」或按量阶梯价 |
| **对账与运营** | 简单，便于跑通流程 | 需系统化对账与配额管理 |

### 2.2 推荐合作模式（10 家）

- **按试衣次数向商家收费**  
  - 约定：用户试穿该商家款式并成功生成 1 张图，计 1 次；平台按次向该商家收费。  
  - **单次报价建议 ≥ 0.8～1.0 元**（覆盖 0.55 元成本 + 平台毛利），具体与商家谈判确定。  
- **保底 + 按量（可选）**  
  - 例如：每月保底 200 元，对应 300 次试衣；超出部分按 0.9 元/次计费。  
  - 保证平台在「试衣量未达预期」时也有最低收入。  
- **数据反馈**  
  - 向商家提供：其款式被试穿次数、从平台跳转至店铺的点击/成交（若可追踪），便于续约与扩量。

**10 家时的粗算（仅作理解用）**  
- 假设每家每月带来 400 次试衣，共 4 000 次/月。  
- 成本：4 000 × 0.55 = 2 200 元。  
- 若按 0.9 元/次向商家收费：4 000 × 0.9 = 3 600 元 → 毛利约 1 400 元/月（未计其他成本）。  
- 商家侧：每家约 360 元/月，可接受度需实地沟通。

### 2.3 推荐合作模式（20 家）

- **仍以「按试衣次数」为主**，便于与 0.55 元/张对齐、不亏本。  
- **包月 + 超额按量**更易谈：  
  - 例如：月费 150 元含 200 次试衣，超出按 0.85 元/次。  
  - 20 家 × 150 = 3 000 元保底；若总试衣 8 000 次，超额 4 000 次 × 0.85，收入增加，总成本 8 000 × 0.55 = 4 400 元，由 3 000 + 超额收入覆盖。  
- **阶梯价（可选）**：量越大单次价略降，激励商家多推款、多引流。  
- 20 家时需**明确配额与归属**：每次试衣归属哪家商家（按所选款式/衣库关联），用于对账与计费。

### 2.4 小结

- **10 家**：重点验证「按次计费 / 保底+按量」能否覆盖 0.55 元并小有盈利；合作方式简单，便于迭代。  
- **20 家**：在 10 家跑通基础上扩量，用包月+按量或阶梯价平衡商家承受度与平台毛利。  
- **无论几家，原则一致**：**每张生成图都对应到某商家，由该商家的合作费覆盖该次 0.55 元成本**，平台不亏本。

## 三、激励制度与产品操作的结合

目标：**用「时尚能量 + 连续登录 + 积分」既提升留存与可玩性，又控制试衣次数，避免无上限免费导致亏本**；**不向用户收钱**。

### 3.1 时尚能量（与试衣强绑定）

| 项目 | 说明 |
|------|------|
| **含义** | 一种「次数型货币」：主要用于**兑换试衣次数**（生成 1 张图 = 消耗一定能量）。 |
| **与操作的结合** | **每成功生成 1 张试衣图，扣除固定时尚能量**（建议 **10 点/张**，可配置）。用户能量不足时，无法发起试衣生成，需通过获取能量后再试。 |
| **每日试衣次数目标** | **用户每天至少可有 5 次以上试衣**：通过「每日登录 + 连续登录加成」发放约 **55～60 点/天**，按 10 点/张计，约 **5～6 次/天**；再通过「平台购货、投稿采纳、抖音获赞」等任务额外获得能量，可超过 5 次。 |
| **获取方式** | **仅通过行为获取，不卖钱：** |
| | • **每日登录**：当日首次登录发放，基础 **30** 点； |
| | • **连续登录加成**：连续第 1 天 +5，第 2 天 +10，第 3 天 +15，第 4 天 +20，第 5 天及以后 +25（单日最多 30+25=**55 点**）。即仅靠登录即可约 **5 次/天**。 |
| | • **通过本平台链接购买商品**：用户从平台跳转至商家并完成下单购买，经对账/回调确认后，发放时尚能量（建议 **15～30 点/单**，可按订单金额阶梯，可配置）。既激励「试穿后去买」，又让平台从 CPS 分成中覆盖成本。 |
| | • **投稿搭配被采纳**：用户提交的搭配经审核采纳后，除积分外**同时发放时尚能量**（建议 **20 点/次**，可配置）。鼓励 UGC 与衣库内容供给。 |
| | • **发布作品到抖音且累计获赞 ≥10**：用户将试衣出片等发布到抖音，累计获赞达到 10 及以上后，在平台内提交截图/链接核销，审核通过后发放时尚能量（建议 **25 点/次**，可配置；同一作品仅可核销 1 次，防刷）。既拉新又促传播。 |
| **设计意图** | 登录即能保证**每日 5 次以上**试衣；购货、投稿、抖音任务提供**额外能量**，把「用平台」和「买商品、发内容」绑定，提升转化与传播，同时成本由 B 端与 CPS 覆盖。 |

**建议在界面上的体现**  
- 试衣页：展示当前时尚能量；点击「生成」时校验能量 ≥ 10，不足则提示「时尚能量不足，去登录/购货/投稿/发抖音获取更多」并引导任务入口。  
- 生成成功后：扣减 10 点，并提示「已消耗 10 点时尚能量」。  
- 「我的」或任务中心：明确列出「通过平台链接购买得能量」「投稿被采纳得能量」「抖音获赞≥10 核销得能量」的规则与入口。

### 3.2 连续登录

| 项目 | 说明 |
|------|------|
| **含义** | **连续登录天数**：连续多少天「当日有过登录」。 |
| **与操作的结合** | **不直接兑换试衣**，而是通过**提高当日登录获得的时尚能量**间接增加试衣能力（见上）。 |
| **规则** | 每日任意一次登录计为当日已登录；中断则从 1 重新计。同一天多次登录不重复增加天数。 |
| **可扩展** | 连续 7 天额外送积分、连续 30 天送专属标签或解锁款等，与「不向用户收费」不冲突。 |

### 3.3 积分

| 项目 | 说明 |
|------|------|
| **含义** | 用于**积分解锁衣库中「需积分才能试穿」的搭配**，解锁后**永久可试穿**（仍消耗时尚能量生成图，不额外扣积分）。 |
| **与操作的结合** | • **获得**：投稿被采纳（如 10 分/次）、完成指定任务（如分享、连续登录档位奖励）等；**不以卖积分包为主**，保持「做任务得积分」的玩法。  
| | • **消耗**：在衣库中点击「积分解锁」某套搭配，扣减对应积分，解锁后该款对用户永久开放。 |
| **与成本的关系** | 积分**不兑换试衣次数**，只解锁款式；试衣次数仍由时尚能量控制，避免「积分无限换试衣」导致成本失控。 |

### 3.4 三者关系小结

- **时尚能量**：和「生成一张图」一一对应，**生成 1 张 = 扣 10 点**；来源为**登录 + 连续登录（约 5 次/天）+ 平台购货 + 投稿采纳 + 抖音获赞≥10 核销**，**不售卖**。  
- **连续登录**：提高当日登录获得的时尚能量，进而保证**每日至少约 5 次**试衣能力。  
- **积分**：只影响**能试穿哪些款**（解锁衣库），**不影响能生成几张图**；生成张数由能量控制。

这样，**用户侧完全不用付钱**，且**每天至少有 5 次以上试衣**，额外可通过购货、投稿、抖音任务获得更多能量；总成本由 **B 端按次/按约 + CPS 分成** 覆盖。

## 四、时尚能量与积分的「用处」（用户视角）

- **时尚能量的用处**  
  - **兑换试衣次数**：有能量才能点「生成」；能量越多，能生成的张数越多。**每天登录即有约 5 次以上**，还可通过**平台购货、投稿被采纳、抖音获赞≥10 核销**获得更多。  
  - 后续可扩展：兑换「免积分解锁某款一次」、优先生成排队等，均不直接收用户钱。

- **积分的用处**  
  - **解锁衣库中需积分的搭配**：部分款式标注「需 20 积分」等，用户用积分解锁后，该款永久可选（每次试衣仍扣时尚能量）。  
  - 形成「投稿/任务 → 积分 → 解锁更多款 → 更多可试穿」的正向循环，不涉及付费。

## 五、不向用户收钱，怎么赚钱？与淘宝、豆包等的差异

### 5.1 为何不向用户收钱

- 若对「生成一张图」向用户收费（尤其 ≥0.55 元），会与**淘宝等购物平台**（多数免费浏览、免费试穿插件）和**豆包等免费/低价生成图片**形成直接对比，**缺乏价格与心理优势**。  
- 用户更容易选择「免费豆包 + 自己找图」或「淘宝内试穿」，我们难以在「单次生成价格」上胜出。  

因此选择：**C 端不向用户收钱，用激励规则控制用量；收入主要来自 B 端。**

### 5.2 怎么赚钱（收入从哪来）

| 来源 | 说明 |
|------|------|
| **B 端商家** | **主要收入**。用户试穿某商家款式并生成 1 张图，平台向该商家收取 ≥0.55 元/次（建议 0.8～1.0 元），或通过保底+按量、包月+超额按量覆盖成本并留毛利。 |
| **C 端** | **不依赖向用户收费**。可保留「极轻量」增值（如少量会员专属款式、去水印等），但**不把「试衣次数」作为主要付费点**，避免与豆包/淘宝比价。 |
| **数据/广告（可选）** | 后期在合规前提下，可为商家提供脱敏洞察或品牌合作，作为补充，不作为早期假设。 |

**结论**：**赚钱靠 B 端合作（10 家、20 家的按次/保底/包月），不靠向用户卖试衣次数或卖积分包**；用户侧「免费 + 能量（每日 5+ 次 + 任务额外）」即可。

---

### 5.4 更多盈利方式（扩展清单）

在「B 端按次/保底/包月」为主的前提下，以下方式可作为补充，按阶段选用：

| 方式 | 说明 | 阶段建议 |
|------|------|----------|
| **CPS 带货分成** | 用户通过平台跳转至商家并下单，平台与商家约定按成交额比例（如 5%～15%）分成。用户「通过平台链接购买」同时获得时尚能量，平台用分成覆盖试衣成本与激励成本。 | 与 10 家/20 家合作同步推进 |
| **按试衣次数向商家收费** | 已述：每试衣 1 次向对应商家收 0.8～1.0 元。 | 主收入，全程 |
| **按点击到店收费（CPC）** | 用户从平台点击进入商家店铺/商品页即计费，与商家约定单次点击价。适合「先曝光、后转化」的商家。 | 商家可选，与按次/CPS 组合 |
| **品牌/联名/活动合作** | 某品牌或活动方支付合作费，在衣库中做专属位、主题活动或联名款露出。一次性或按季/按年签约。 | 有案例后拓展 |
| **数据与洞察服务** | 向商家出售脱敏后的「款式被试穿排行、风格偏好、转化路径」等报告或看板，按年/按次收费。 | 中后期，数据积累后 |
| **信息流/品牌广告** | 在推荐流、衣库列表等位置接入与穿搭/服饰相关的品牌广告，按 CPM 或 CPC 计费。需兼顾体验与合规。 | 中后期，用户量起来后 |
| **会员（轻量）** | 会员享受：免广告、专属款式、每月额外时尚能量包、去水印出片等。定价不宜高，作为补充而非主收入。 | 验证留存后可上 |
| **商品图/内容制作服务** | 面向商家：用平台试衣能力批量生成商品图、搭配图、模特图，按张或按套餐收费。与「按试衣次数」区分，属 To B 定制。 | 与 1688/社媒招商结合 |
| **培训/入驻/招商费** | 对商家收取一次性入驻费或年度服务费，含培训、对接、数据看板等。适合中后期标准化运营时。 | 后期 |
| **与抖音等平台合作** | 用户发抖音带平台话题/链接并获赞≥10 可核销得能量，平台可争取与抖音的流量扶持、分成或品牌合作，换曝光或资源。 | 有量后可谈 |

**优先建议**：早期以 **B 端按次 + CPS 分成** 为主（用户购货即带来 CPS，同时发能量激励）；再视数据增加 **CPC、品牌合作、数据服务**；**会员与广告**作为补充，不喧宾夺主。

### 5.3 与淘宝、豆包的差异化（简要）

- **淘宝等**：重在购物与比价，试穿多为附属；我们做**穿搭场景 + 天气/场合推荐 + 衣库可购 + 试衣与激励闭环**，差异化在「场景与流程」，不是比单次试衣价格。  
- **豆包等**：单次生成免费或便宜，但无现成衣库、无款款可购、无推荐与投稿/积分闭环；我们强调**即开即用、可购、可玩、可成长**，用户不付钱也能用，但体验和闭环不同。  

因此：**不向用户收钱，反而有利于和淘宝、豆包形成差异化**；盈利由 B 端保障。

## 六、对《项目计划书》第七部分的修改建议

以下建议可直接用于修订《项目计划书-项目概述.md》中的**第七部分 盈利与商业化**，或在该部分增加「详见《激励制度与盈利方案-单独文档》」的引用。

### 6.1 建议修改/强化的点

1. **试衣成本与不亏本（第七部分（一））**  
   - 保留「0.55 元/张」及「收入覆盖成本」的表述。  
   - **明确**：平台通过 **B 端按次/保底/包月收费覆盖每张图 0.55 元**；C 端通过**时尚能量限制试衣次数**，不依赖对用户收费覆盖成本。

2. **盈利来源（第七部分（二））**  
   - **B 端**：列为首要来源；说明「用户试穿某商家款式生成 1 张图，即向该商家计费 1 次，单次建议 ≥0.8 元」。  
   - **C 端**：改为「以免费 + 激励为主；不以试衣次数或积分包为主要收费点；可选极轻量会员/增值」。

3. **B 端合作模式（第七部分（三））**  
   - 补充：**10 家**阶段以「按次计费或保底+按量」为主，验证单位经济性；**20 家**阶段可引入「包月+超额按量」或阶梯价，并说明试衣归属与对账方式。  
   - 注明：具体单价与档位以单独文档《激励制度与盈利方案-单独文档》为准。

4. **激励与用量的关系（可放在第七部分或单独小节）**  
   - 增加一句：**激励制度（时尚能量、连续登录、积分）与产品操作的对应关系及「不向用户收费」下的成本控制，见单独文档《激励制度与盈利方案-单独文档》。**

   - 在第七部分或竞品部分点明：**为避免与淘宝、豆包等在「单次生成价格」上直接竞争，采用 C 端免费 + B 端收费的模式，收入以 B 端为主。**
