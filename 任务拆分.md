# 穿搭应用 - 任务拆分（便于 Cursor 实现）

## 项目概览

- **目标**：虚拟试衣 + 智能穿搭推荐 + 积分/解锁 + 社交分享
- **技术方向**：前端（Web/小程序）+ 后端 API + AI（图像生成/客服/推荐）

---

## 一、功能 1：虚拟试衣（上传照片 → 三视图 + 360° 视频）

### 1.1 用户与体型建模

| 序号 | 任务 | 说明 | 依赖 |
|------|------|------|------|
| 1.1.1 | 用户注册/登录与基础信息 | 支持手机/邮箱登录，存储用户 ID | 无 |
| 1.1.2 | 用户体型表单 | 表单字段：性别、身高、体重、体型选择 | 1.1.1 |
| 1.1.3 | 体型参考图与选项 | 设计/准备几张常见体型参考图（如：偏瘦/标准/微胖/偏胖等），用户可点选 | 无 |
| 1.1.4 | 体型 → 提示词映射 | 将体型选项映射为固定模板提示词（如：避免紧身、适合宽松等） | 1.1.2, 1.1.3 |

### 1.2 衣库与选款

| 序号 | 任务 | 说明 | 依赖 |
|------|------|------|------|
| 1.2.1 | 衣库数据模型 | 定义「一套搭配」的数据结构（名称、图片、风格标签、是否需积分解锁等） | 无 |
| 1.2.2 | 衣库后台/配置 | 支持上传、编辑、上下架搭配；可先做 JSON/表格配置，后续可接 CMS | 1.2.1 |
| 1.2.3 | 衣库展示页 | 列表/卡片展示所有可选搭配，支持按风格筛选；已解锁/未解锁状态展示 | 1.2.1, 积分系统 |

### 1.3 照片上传与生成

| 序号 | 任务 | 说明 | 依赖 |
|------|------|------|------|
| 1.3.1 | 头像/照片上传 | 用户上传一张清晰头像/半身照，校验格式与大小，存储 URL 或 ID | 1.1.1 |
| 1.3.2 | 固定模板提示词组装 | 将：用户体型提示词 + 选中搭配描述 + 固定风格指令 组装成完整 prompt | 1.1.4, 1.2.1 |
| 1.3.3 | 调用图像生成 API | 对接图像生成服务（如 SD、Midjourney API、或国内视觉大模型），传入照片+提示词，生成正面/侧面/背面三视图 | 1.3.1, 1.3.2 |
| 1.3.4 | 三视图展示页 | 展示生成的三张图，支持保存/分享 | 1.3.3 |
| 1.3.5 | 360° 视频生成（可选分阶段） | 多角度图 → 合成旋转视频，或对接支持 360 展示的接口；可先做「多角度图」再迭代视频 | 1.3.3 |

---

## 二、功能 2：智能客服

### 2.1 对话与问答

| 序号 | 任务 | 说明 | 依赖 |
|------|------|------|------|
| 2.1.1 | 客服对话界面 | 聊天 UI：文本 + 支持上传图片（搭配图） | 无 |
| 2.1.2 | 客服 LLM 对接 | 接入大模型 API，系统提示词包含：平台衣库介绍、常见问题、购物平台说明 | 2.1.1 |
| 2.1.3 | 服装问答与购物链接 | 用户上传搭配图并问「哪里买」时，识别风格/单品并返回常用平台购物链接（可配置链接模板） | 2.1.2, 衣库/商品数据 |
| 2.1.4 | 转人工入口 | 按钮「转人工」，接入开发者会话（如客服后台、或简单工单） | 2.1.1 |
| 2.1.5 | 开发者不在时的 fallback | 不在线时提示「留言」或「稍后回复」，并记录留言 | 2.1.4 |

### 2.2 用户投稿与积分

| 序号 | 任务 | 说明 | 依赖 |
|------|------|------|------|
| 2.2.1 | 用户上传自荐搭配 | 在客服中支持「投稿我的搭配」：上传图 + 简短描述，落库待审核 | 2.1.1 |
| 2.2.2 | 积分账户与规则 | 用户积分表；规则：投稿被采纳 → 增加 N 积分 | 用户系统 |
| 2.2.3 | 采纳流程 | 后台标记「采纳」→ 触发积分发放；可选通知用户 | 2.2.1, 2.2.2 |
| 2.2.4 | 积分解锁服装 | 衣库中部分搭配需消耗积分解锁，校验积分并扣减 | 1.2.1, 2.2.2 |

---

## 三、功能 3：快速穿搭推荐

| 序号 | 任务 | 说明 | 依赖 |
|------|------|------|------|
| 3.1 | 天气获取 | 接入天气 API（按用户定位或选择城市），得到温度、阴晴等 | 无 |
| 3.2 | 场合选择 | 选项：约会/日常/通勤/运动等，可多选或单选 | 无 |
| 3.3 | 推荐逻辑 | 根据天气 + 场合筛选衣库搭配，返回 3 套；可规则引擎或简单标签匹配 | 1.2.1, 3.1, 3.2 |
| 3.4 | 三套展示与交互 | 展示 3 套搭配卡片；支持「刷新」重新推荐、「随机抽卡」随机一套 | 3.3 |
| 3.5 | 从推荐到试衣 | 点击某套 → 跳转到功能 1 的试衣流程（选照片+生成三视图） | 1.2.3, 1.3.x |

---

## 四、功能 4：用户画像（穿搭风格总结）

| 序号 | 任务 | 说明 | 依赖 |
|------|------|------|------|
| 4.1 | 穿搭记录存储 | 用户每次「选择/生成」的搭配落库，带时间、场合、天气等 | 1.2, 3.x |
| 4.2 | 风格分析逻辑 | 定期或按需对历史记录做汇总：颜色偏好、风格标签、场合分布等 | 4.1 |
| 4.3 | 用户画像展示页 | 展示「你的穿搭风格」：关键词、占比、简单图表；可引用 4.2 的结果 | 4.2 |

---

## 五、功能 5：抖音分享与解锁

| 序号 | 任务 | 说明 | 依赖 |
|------|------|------|------|
| 5.1 | 分享到抖音的引导 | 生成带水印/文案的图片或短链，引导用户复制链接或保存图发抖音 | 1.3.4 |
| 5.2 | 点赞数核销 | 用户把抖音链接/截图发给客服，人工或 OCR/链接解析 核验点赞≥10 | 2.1 |
| 5.3 | 解锁奖励 | 核验通过后，为用户解锁「一套穿搭」的试衣权限（同 2.2.4 的解锁逻辑） | 2.2.4, 5.2 |

---

## 六、通用与基础设施（建议优先或并行）

| 序号 | 任务 | 说明 |
|------|------|------|
| 6.1 | 技术栈选型 | 前端：React/Vue/小程序 选一；后端：Node/Python/Go 选一；数据库：PostgreSQL/MySQL + 对象存储 |
| 6.2 | 用户与权限 | 统一用户 ID、登录态、基础权限（是否可试衣、积分余额等） |
| 6.3 | 衣库与积分数据模型 | 见 1.2.1、2.2.2；先定表结构再开发功能 |
| 6.4 | 部署与环境 | 开发/测试环境；图像生成与 LLM 的 API Key 管理（环境变量） |

---

## 建议实施顺序（按 Cursor 可迭代小块）

1. **阶段 0**：6.1 技术栈 + 6.2 用户 + 6.3 数据模型（表结构 + 衣库 JSON）
2. **阶段 1**：1.1 体型 + 1.2 衣库展示 + 1.3.1~1.3.4（上传照片 → 固定模板提示词 → 三视图生成与展示）
3. **阶段 2**：2.1 智能客服（对话 + 转人工）+ 2.2 积分与投稿 + 衣库「需解锁」与积分扣减
4. **阶段 3**：3.x 快速穿搭（天气 + 场合 + 三套推荐 + 刷新/随机）
5. **阶段 4**：4.x 用户画像（记录 + 分析 + 展示页）
6. **阶段 5**：5.x 抖音分享与点赞解锁；1.3.5 的 360° 视频可放在最后或单独迭代

每个子任务都可以在 Cursor 里单独开一个需求（如：实现用户体型表单 + 体型到提示词的映射），便于一步步完成。
