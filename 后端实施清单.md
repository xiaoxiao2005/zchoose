# 后端你需要做什么

## 一、环境与工具准备（你要先做的）

1. **安装 Node.js**  
   建议 LTS 版本（如 20.x）。[下载](https://nodejs.org/) 后确认：
   ```bash
   node -v
   npm -v
   ```

2. **数据库**  
   当前使用 **SQLite**（通过 sql.js 纯 JS 实现），无需单独安装数据库或 Windows 编译环境，`npm install` 即可。

3. **准备后续会用到的 Key（先不写进代码）**  
   - 图像生成 API Key（如国内大模型视觉 API、或 Stable Diffusion 服务）。  
   - 智能客服用的大模型 API Key。  
   - 天气 API Key（如和风、心知等）。  
   先在 `.env` 里留好变量名，等对接时再填。

## 三、后端「AI 图片美化」预留（与前端界面无关）

- **存图策略**：AI 生成的三视图建议落库并保留「原始生成图」ID，方便后续对图片做算法级美化。
- **接口预留**：`POST /api/try-on/:resultId/beautify`，请求体可带 `{ type: 'skin_smooth' | 'color_tune' }`，实现时再接修图 API 或自建模型。

## 五、前后端基本搭建完成后请配置 API

后端与前端骨架都就绪后，请记得在 **`backend/.env`** 中配置以下 Key（参考 `backend/.env.example`），否则对应功能为 mock/占位：

| 用途 | 环境变量 | 说明 |
|------|----------|------|
| 图像生成（试衣三视图） | `IMAGE_GEN_API_KEY`、`IMAGE_GEN_BASE_URL` | 在 `tryOn` 生成逻辑中替换 mock，写入真实三视图 URL |
| 智能客服 LLM | `LLM_API_KEY`、`LLM_BASE_URL` | 在 `support/chat` 中替换占位回复 |
| 天气 | `WEATHER_PROVIDER=qweather` + `WEATHER_API_KEY` 或 `WEATHER_API_URL` + `WEATHER_API_KEY` | 和风天气或自定义单 URL |
| JWT | `JWT_SECRET`、`JWT_EXPIRE` | 登录签发 token，生产环境务必修改 `JWT_SECRET` |

配置完成后，重启后端服务使 `.env` 生效。
