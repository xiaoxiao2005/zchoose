# 登录/注册（手机号）自测指南

## 一、前置条件

1. **启动后端**（终端 1）  
   ```bash
   cd d:\cursor\backend
   npm run dev
   ```
   看到 `后端运行在 http://localhost:3000` 即正常。

2. **启动前端**（终端 2）  
   ```bash
   cd d:\cursor\frontend
   npm run dev
   ```
   浏览器访问 **http://localhost:5173**（若 Vite 代理已配好，请求会转发到 3000）。

3. 确认前端代理已指向后端（`frontend/vite.config.ts` 中应有 `proxy: { '/api': 'http://localhost:3000' }`），否则需在请求里写完整后端地址。

## 二、后端接口自测（可选）

用 PowerShell 或 curl 直接调接口，验证逻辑与错误码。

### 1. 发送验证码

```powershell
# 替换成你的 11 位手机号
$body = '{"phone":"13800138000"}'
Invoke-RestMethod -Uri "http://localhost:3000/api/users/send-code" -Method POST -Body $body -ContentType "application/json"
```

- **预期**：返回 `message: "验证码已发送"`，且开发环境下会带 `code`（6 位数字）。
- **错误示例**：不传 `phone` 或传空 → 400；非 11 位手机号 → 400。

### 2. 注册

先用上面接口拿到 `code`，再调用注册（同一手机号 + 该验证码 + 密码）：

```powershell
# 把 123456 换成上一步返回的 code，111111 为密码（至少 6 位）
$body = '{"phone":"13800138000","code":"123456","password":"111111"}'
Invoke-RestMethod -Uri "http://localhost:3000/api/users/register" -Method POST -Body $body -ContentType "application/json"
```

- **预期**：返回 `token`、`userId`、`phone`。
- **错误示例**：验证码错误/过期 → 400；该手机号已注册 → 409；密码不足 6 位 → 400。

### 3. 登录

```powershell
$body = '{"phone":"13800138000","password":"111111"}'
Invoke-RestMethod -Uri "http://localhost:3000/api/users/login" -Method POST -Body $body -ContentType "application/json"
```

- **预期**：返回 `token`、`userId`、`phone`。
- **错误示例**：手机号未注册或密码错误 → 401。

## 三、前端页面自测（推荐）

在浏览器打开 **http://localhost:5173**，按下面步骤操作。

### 1. 打开登录弹窗

- 点击顶栏右侧 **「登录」**。
- **预期**：弹出登录/注册弹窗，有「登录」「注册」两个 Tab，仅「手机号」输入框（无邮箱/手机切换）。

### 2. 注册流程

1. 切到 **「注册」** Tab。
2. 输入 11 位手机号（如 `13800138000`）。
3. 点击 **「获取验证码」**。
   - **预期**：按钮进入 60 秒倒计时；开发环境下若后端返回了 `code`，可看网络响应或控制台拿到验证码（或看后端终端日志）。
4. 输入验证码、密码（至少 6 位）、确认密码一致。
5. 可勾选「是否记住密码」。
6. 点击 **「注册」**。
   - **预期**：弹窗关闭，顶栏显示该手机号与「退出」，表示已登录。

### 3. 登录流程（刚注册的账号）

1. 若已登录，先点 **「退出」**。
2. 再点 **「登录」**，在 **「登录」** Tab 输入同一手机号和密码，点击 **「登录」**。
   - **预期**：弹窗关闭，顶栏显示手机号，表示登录成功。

### 4. 记住密码

1. 退出后再次打开登录弹窗，在登录 Tab 勾选「记住密码，下次直接登录」，再登录一次。
2. 退出，关闭浏览器（或刷新页面），再次打开登录弹窗。
   - **预期**：手机号和密码已预填，可直接点登录。

### 5. 个人页展示

- 登录后进入 **「我的」** 页。
   - **预期**：个人信息里显示「手机号：13800138000」（或你用的号码），无邮箱项。

### 6. 错误与边界

| 操作 | 预期 |
|------|------|
| 注册时手机号非 11 位或格式不对 | 提示「请输入正确的 11 位手机号」 |
| 注册时不点获取验证码或验证码错误 | 提示「请先获取并填写验证码」或「验证码错误」 |
| 注册时两次密码不一致 | 提示「两次输入的密码不一致」 |
| 注册时密码少于 6 位 | 提示「密码至少 6 位」 |
| 用已注册手机号再注册 | 提示「该手机号已注册」 |
| 登录时密码错误 | 提示「账号或密码错误」 |
| 登录时手机号未注册 | 提示「账号或密码错误」 |

## 四、自测检查清单（可打勾）

- [ ] 后端 send-code 仅手机号，返回 code（开发环境）
- [ ] 后端 register 仅手机号+验证码+密码，成功返回 token
- [ ] 后端 login 仅手机号+密码，成功返回 token
- [ ] 前端弹窗只有手机号，无邮箱/忘记密码
- [ ] 注册：获取验证码 → 填码+密码+确认密码 → 注册成功并登录
- [ ] 登录：手机号+密码 → 登录成功，顶栏显示手机号
- [ ] 退出后再次登录正常
- [ ] 记住密码后，再次打开弹窗有预填
- [ ] 我的页显示「手机号」而非「邮箱」
- [ ] 错误提示符合上表（验证码错误、密码不一致、已注册等）

按上述步骤跑一遍，即可完成登录/注册这部分的 self-test。
