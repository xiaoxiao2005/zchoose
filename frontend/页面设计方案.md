# Zchoose 穿搭应用 - 前端页面设计方案（游戏化横屏版）

用于前后端思路统一。**应用名：Zchoose**。**操作取向：横屏**。**整体风格：游戏感布局**（参考王者等：左右分栏、左侧按钮弹出功能、右侧大块展示、顶/底栏、滑动与动效）。

---

## 〇、启动闪屏（打开应用第一屏）

**用途**：用户点击/打开应用时，第一眼看到的画面，建立品牌印象。

**动效与节奏**：

1. **黑屏**：全屏纯黑（#000 或接近黑），持续约 0.3～0.5 秒。
2. **闪亮**：屏幕由黑逐渐亮起（渐变或轻微光晕扩散），约 0.5 秒。
3. **露出品牌名**：在亮起过程中或亮起后，中央出现应用名 **「Zchoose」**（可带简单 logo 或发光描边）。
4. **停留**：品牌名保持约 1～1.5 秒（可配合轻微呼吸光效或静态）。
5. **进入首页**：淡出或短过渡后跳转至首页（横屏主界面）。

**布局示意**：

```
┌──────────────────────────────────────────────────────────────────┐
│                                                                  │
│                          ( 黑屏 )                                 │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
                              ↓ 闪亮
┌──────────────────────────────────────────────────────────────────┐
│                                                                  │
│                                                                  │
│                         Z c h o o s e                             │  ← 居中、可发光/描边
│                                                                  │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

**技术要点**：单页或独立路由（如 `/splash`），计时/动画结束后 `replace` 到首页；横屏提示可放在闪屏之后或首次进入时提示「建议横屏使用」。

## 一、整体结构（横屏 + 游戏式布局）

### 1.1 应用形态

- **Web 单页应用（SPA）**，路由区分页面。
- **横屏为主**：默认按横屏设计，主内容区域按 16:9 或宽屏比例适配；竖屏时可提示「请横屏以获得最佳体验」或提供竖屏降级布局（可选）。
- **游戏感**：参考王者等手游大厅——**左侧窄条为功能入口（图标按钮，点击弹出面板或切换右侧内容）**，**右侧为主展示区（大块内容、列表、详情、3D/大图等）**，**顶部栏（Logo、状态、用户）**，**底部栏（主玩法/主入口或全局导航）**；支持**滑动**切换、面板滑入滑出等动效。

### 1.2 全局 shell 布局（横屏）

整体采用「左 + 右 + 顶 + 底」四分结构：

```
┌─────────────────────────────────────────────────────────────────────────────┐
│ 顶栏 [Zchoose Logo]  [积分/通知]  [用户头像/登录]  [设置]                      │  ← 顶部导航栏
├──────┬──────────────────────────────────────────────────────────────┬───────┤
│ 大厅 │                                                              │       │
│ 衣库 │                    右 侧 主 展 示 区                           │ 可选  │
│ 快速 │  （当前页面：今日推荐 / 衣库列表 / 试衣 / 客服 / 我的 等）             │ 右栏  │
│ 穿搭 │  点击左侧任一项 → 跳转对应界面，右侧切换为该项内容                    │ 或无  │
│ 试衣 │                                                              │       │
│ 客服 │                                                              │       │
│ 我的 │                                                              │       │
├──────┴──────────────────────────────────────────────────────────────┴───────┤
│ 底栏 [ 首页 ]  [ 衣库 ]  [ 快速穿搭 ]  [ 客服 ]  [ 我的 ]                      │  ← 底部主导航
└─────────────────────────────────────────────────────────────────────────────┘
```

- **顶部栏**：左侧 Zchoose Logo（可点击回首页）；右侧：积分、消息红点、用户头像/登录、设置。风格可偏深色/半透明，游戏 HUD 感。
- **左侧功能钮（首页）**：**首页左侧 = 所有功能入口**，竖排一排按钮，每项对应一个主界面；**点击任一侧边按钮即跳转至对应界面**（与底栏导航一致，右侧主区切换为该项内容）。详见下节「首页左侧全部功能」。
- **右侧主展示区**：当前页核心内容，占满中间大块；可内部分区（如顶部 Tab + 下方列表）、支持左右滑动切换卡片或列表。
- **底部栏**：固定 5 个主入口——**首页 / 衣库 / 快速穿搭 / 客服 / 我的**，与当前路由高亮一致；风格与顶栏统一，图标+文字。底栏与首页左侧均可导航，**首页左侧在首屏集中展示全部功能**，便于一眼看到并跳转。

### 1.3 导航与入口对应

| 入口名称 | 说明           | 点击后右侧主区内容           |
|----------|----------------|-----------------------------|
| 首页/大厅 | 主入口         | 今日推荐 + 天气 + 双 CTA     |
| 衣库     | 全部搭配       | 衣库列表（筛选 + 卡片网格）  |
| 快速穿搭 | 天气+场合推荐  | 三套推荐 + 刷新/随机         |
| 试衣     | 虚拟试衣流程   | 选款 → 上传照片 → 生成三视图 |
| 客服     | 智能客服       | 对话列表 + 输入区            |
| 我的     | 个人中心       | 积分、体型、投稿、核销、画像 |

**首页左侧按钮 = 以上全部 6 项**（大厅、衣库、快速穿搭、试衣、客服、我的），竖排排列，点击即跳转至对应界面；当前所在界面可在左侧高亮。底栏保留 5 项（首页、衣库、快速穿搭、客服、我的），试衣可从「首页」内进入或从左侧「试衣」直达。

### 1.4 全局约定

- **登录态**：试衣、投稿、解锁、积分等需登录；未登录时点击相关入口可弹出登录层或跳转登录页。
- **品牌名**：全局统一使用 **Zchoose**（启动页、顶栏 Logo、标题等）；所有文案/主色/Logo 处保留 `【界面美化-可替换】` 批注。
- **横屏**：主要布局按横屏设计；若做竖屏适配，可保留顶栏+底栏，中间单列或折叠左侧为顶部下拉。

## 二、各页面设计方案（在 shell 内嵌）

以下各页均为「顶栏 + 左侧栏 + 右侧主区 + 底栏」下的**右侧主区内容**（及左侧/弹层配合方式）。

### 2.1 首页（大厅）

**用途**：横屏下的主入口，突出「试衣」与「快速穿搭」，带游戏感左右布局与滑动。

**右侧主区布局示意**：

```
┌─────────────────────────────────────────────────────────────────┐
│  今日 22°C 晴 · 北京                    [ 换一换 ]  [ 随机抽一套 ] │  ← 天气 + 快捷操作
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────┐  ┌─────────────┐  ┌─────────────┐             │
│   │   搭配 1    │  │   搭配 2    │  │   搭配 3    │   ← 三张卡片  │
│   │   [ 大图 ]  │  │   [ 大图 ]  │  │   [ 大图 ]  │     可左右滑动 │
│   │   名称      │  │   名称      │  │   名称      │             │
│   │ [ 去试穿 ]  │  │ [ 去试穿 ]  │  │ [ 去试穿 ]  │             │
│   └─────────────┘  └─────────────┘  └─────────────┘             │
│         ·  ·  ·   ← 指示点（当前第几组）                          │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│  [ 立即试衣 ]  —  选搭配 · 上传照片 · 生成三视图                  │
│  [ 快速穿搭 ]  —  根据天气与场合，一键出三套                      │
└─────────────────────────────────────────────────────────────────┘
```

**左侧按钮（首页）**：**包含所有功能入口**，竖排一排，点击即跳转对应界面：

| 左侧按钮 | 点击后跳转 |
|----------|------------|
| 大厅     | 当前首页（今日推荐 + 双 CTA） |
| 衣库     | 衣库列表页 |
| 快速穿搭 | 快速穿搭页（天气 + 场合 + 三套） |
| 试衣     | 虚拟试衣流程（选款 → 上传 → 生成） |
| 客服     | 智能客服对话页 |
| 我的     | 个人中心（积分、体型、投稿、核销、画像） |

- 每项为图标 + 文字（或仅图标，hover/展开显示文字），当前所在界面在左侧高亮。
- **仅做跳转**：点击后右侧主区切换为该项对应页面内容，路由与底栏同步（底栏选中项与左侧选中项一致）。

**主要模块**：

- **天气 + 操作**：顶部一小条展示城市、温度、天气现象（`GET /api/weather`）；右侧「换一换」「随机抽一套」对应刷新推荐、随机一套（`GET /api/recommend`，random=1 时抽一套）。
- **今日推荐三卡**：`GET /api/recommend?city=xxx&occasion=日常` 取 3 套，**横排大卡片**，支持**左右滑动**切换（可多组，滑动到下一组 3 张）；每张卡片：大图、名称、「去试穿」按钮；点击「去试穿」带 outfitId 进试衣流程，并可选 `POST /api/recommend/record`。
- **底部双 CTA**（可选保留）：「立即试衣」「快速穿搭」与左侧「试衣」「快速穿搭」作用一致，点击后同样跳转对应界面；可保留作右侧主区内的快捷入口。

**关键交互**：

- 左右滑动：推荐卡片可滑动浏览多组，底部小圆点指示当前组。
- **左侧全部功能按钮**：点击「大厅/衣库/快速穿搭/试衣/客服/我的」任一项 → **跳转至对应界面**，右侧主区切换为该项页面内容；当前所在界面在左侧高亮，与底栏选中项同步。
- 顶栏、底栏、左侧栏始终存在，仅右侧主区随导航变化。

**接口**：`GET /api/weather`、`GET /api/recommend`、`POST /api/recommend/record`。

### 2.4 虚拟试衣流程

**用途**：填写/确认身高、体重、体型及特征提示词 → 上传头像照 → 选穿搭 → 生成三视图或 360° 环绕图；**体型与身高体重等数据与「我的」个人信息关联**，试衣时优先使用「我的」中已保存的数据以生成更贴合的结果。

**流程**：

1. **体型与身体参数**：展示**体型参考图片**（偏瘦/标准/微胖/偏胖），用户选择体型；输入**身高**、**体重**；可输入**其他特征提示词**（如肩宽、腿长等，用于生成更符合本人的图）。若用户已在「我的」中填写过，此处**预填并展示**，用户可修改后用于本次试衣（不强制回写「我的」）。
2. **上传头像图片**：上传一张清晰半身/头像照。
3. **选择穿搭**：从衣库选一套（可带 outfitId 进入则已选）。
4. **生成**：选择「生成三视图」或「生成 360° 环绕图」；生成中 loading，完成后跳结果。
5. **结果**：三视图页展示正面/侧面/背面；若选了 360° 则展示环绕图/视频。底部「保存」「分享到抖音」。

**右侧主区布局示意**：

```
┌─────────────────────────────────────────────────────────────────┐
│  体型参考   [偏瘦] [标准] [微胖] [偏胖]  ← 配参考图，点选           │
│  身高 [____] cm    体重 [____] kg                                │
│  其他特征提示词（选填）[________________________]                  │
│  （以上可从「我的」预填，关联试衣生成更贴合）                       │
├─────────────────────────────────────────────────────────────────┤
│  头像照片   [ 点击上传 ] 或拖拽    已选穿搭 [ 某套名称 ] [ 换一套 ] │
├─────────────────────────────────────────────────────────────────┤
│  [ 生成三视图 ]        [ 生成 360° 环绕图 ]                       │
└─────────────────────────────────────────────────────────────────┘
```

**三视图结果**：右侧主区横排三张大图（正面/侧面/背面），可左右滑动或并排；若为 360° 则展示环绕图/视频。底部「保存图片」「分享到抖音，解锁更多」。

**数据关联**：试衣时使用的身高、体重、体型、其他特征提示词，**优先从「我的」个人信息的体型与身体数据拉取**（GET /api/body-profile/:userId），页面预填；用户可在试衣页本次修改，提交生成时随请求传给后端（height_cm、weight_kg、extra_prompt 等），用于组装提示词，生成更符合的图片。接口：GET/PUT body-profile、GET body-profile/options（含体型参考图 imageUrl）、upload/photo、try-on/generate（body 可带 height_cm、weight_kg、extra_prompt）、try-on/results/:id。  
**体型参考图**：四张图（偏瘦/标准/微胖/偏胖）需在前端搭建好后放入约定路径，详见 `frontend/搭建后待办提醒.md`。

### 2.5 快速穿搭

**用途**：天气 + 场合 → 三套推荐；刷新、随机抽一套；选一套去试衣。

**右侧主区**：顶部城市、场合选择 + 天气展示；中间三张横排大卡（与首页推荐类似）；底部「刷新推荐」「随机抽一套」。选一套「去试穿」→ 带 outfitId 进试衣，并可选 `POST /api/recommend/record`。接口：`GET /api/weather`、`GET /api/recommend`、`POST /api/recommend/record`。

---

### 2.6 智能客服

**用途**：对话（文本+图）、投稿、转人工、抖音核销入口。

**右侧主区**：上方快捷入口「投稿我的搭配」「抖音解锁说明」；中间对话气泡区（可滚动）；底部输入框 + 上传图片 + 发送 +「转人工」。左侧可放「投稿」「抖音核销」图标，点击后右侧切换为对应表单或说明。接口：support/history、support/chat、submissions、douyin/claim、douyin/claims/my。

---

### 2.7 我的（编辑个人信息）

**用途**：**编辑我的个人信息**，与试衣数据关联；展示积分；管理投稿、抖音核销、穿搭画像；登出。

**与试衣的关联**：在「我的」中填写的**性别、体型、身高、体重、其他特征提示词**会保存到后端（body_profile），**试衣生成时自动使用这些数据**组装提示词，生成更贴合用户身材的图片。试衣页会预填「我的」中已保存的数据，用户也可在试衣页当次修改使用。

**右侧主区**：

- **个人信息区**：头像/昵称（或邮箱）；**性别**展示与编辑；**积分**展示（用于解锁衣库）。
- **体型与身体数据**（与试衣关联）：**体型**（偏瘦/标准/微胖/偏胖，可配参考图点选）、**身高**（cm）、**体重**（kg）、**其他特征提示词**（选填，如肩宽、腿长等）。保存后写入 `PUT /api/body-profile/:userId`，试衣时读取并用于生成。
- **下列入口**：我的投稿、抖音核销、我的穿搭风格；底部退出登录。

接口：GET /api/points/:userId、GET/PUT /api/body-profile/:userId（body 含 gender、height_cm、weight_kg、body_type、extra_prompt）、GET /api/body-profile/options、submissions/my、douyin/claims/my、GET /api/profile/:userId/style。

## 三、跨页面与组件复用

- **顶栏 / 底栏**：全局统一组件；顶栏 Zchoose Logo、积分、用户；底栏 5 个主入口，横屏下固定底部。
- **左侧功能栏**：统一「竖排图标 + 点击弹出面板/切换右侧内容」；首页/衣库/客服等可配置不同图标集。
- **搭配卡片**：首页推荐、衣库、快速穿搭共用同一卡片组件（大图、名称、标签、解锁状态、主按钮）。
- **上传图片**：试衣、客服、投稿、抖音共用上传组件（`POST /api/upload/photo`）。
- **启动闪屏**：独立路由或根组件，黑屏 → 闪亮 → Zchoose → 进入首页。

## 四、界面风格与规范（游戏感 + 横屏）

- **主色 / 背景**：深色或半透明 HUD 感（如深灰、深蓝），主按钮可用高亮色（金/青/紫）；CSS 变量 + `【界面美化-可替换】` 批注。
- **横屏**：主布局按横屏比例（如 16:9）设计；左右分栏、大图横滑、底部通栏导航均按横屏优化。
- **动效**：闪屏亮起、面板滑入滑出、卡片滑动、按钮 hover/点击反馈，可适当加强（如轻微缩放、光效）。
- **字体与层级**：标题与数字可略大、加粗；积分、等级等可带描边或发光，增强游戏感。
- **加载与错误**：生成中做 loading 动效；接口失败 toast 或内联提示，可重试。

## 六、实现顺序建议

1. **启动闪屏**：黑屏 → 闪亮 → Zchoose → 跳首页  
2. **全局 shell**：顶栏 + 底栏 + 左侧栏占位，右侧主区占位  
3. **首页**：右侧今日推荐（横滑三卡）+ 天气 + 双 CTA，左侧 2～3 个功能图标  
4. **登录/注册**：弹层或独立页  
5. **衣库**：右侧列表 + 筛选  
6. **试衣流程**：选款 → 体型 → 上传 → 生成 → 结果  
7. **快速穿搭**：场合 + 三套 + 刷新/随机  
8. **客服**：对话 + 投稿/抖音入口  
9. **我的**：积分、体型、投稿、核销、画像  

若你认可这一版（启动闪屏 Zchoose + 横屏游戏式首页与 shell），我们可以按此顺序开始写前端代码；有想改的区块或动效节奏再说即可。
